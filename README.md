# ClientFlow

A modern client and project management platform built for freelancers, consultants, and small agencies.

ClientFlow helps users manage clients, projects, invoices, deadlines, and business activity in one clean and professional dashboard. The project was designed to feel like a realistic business SaaS product rather than a simple CRUD demo.

## Overview

ClientFlow was built to solve a common problem for freelancers and small teams: work is often scattered across spreadsheets, notes, email threads, and invoicing tools. Instead of switching between disconnected systems, ClientFlow brings everything into one place.

The app combines client management, project tracking, task organization, invoicing, and business insights inside a polished, business-focused interface. The main goal of the project was to create a stronger, more serious portfolio piece that demonstrates product thinking, structured data handling, and modern frontend development.

## Key Features

- Client management with searchable client records
- Project tracking with status, budget, progress, and deadlines
- Project detail pages with task organization
- Invoice tracking with paid, unpaid, and overdue states
- Dashboard with business stats, recent activity, and deadline alerts
- Settings page with profile, business information, and notification controls
- Authentication flow with sign up and sign in pages
- Responsive business dashboard layout
- Consistent visual system across pages
- Professional SaaS-style landing page

## Why This Project Stands Out

ClientFlow was intentionally designed to feel more realistic and business-oriented than a typical junior portfolio app.

Instead of focusing only on visual polish or simple task lists, this project emphasizes:

- real-world product structure
- multiple connected data views
- business use cases
- clearer information hierarchy
- stronger UX for dashboards, tables, lists, and management workflows

It represents a step forward from a simpler productivity project into a more serious product environment.

## Getting Started

### Prerequisites

- Node.js 18 or higher
- npm, pnpm, or bun

### Installation

```bash
git clone https://github.com/lazarbukeljovic-dotcom/clientflow.git
cd clientflow
npm install

```

### Run Locally

```bash
npm run dev
```

### Build for Production

```bash
npm run build
npm run preview
```

## Product Thinking Behind the Project

While building ClientFlow, the focus was not only on making the interface look clean, but also on making the app feel believable and useful.

Important decisions included:

- organizing data around clients, projects, invoices, and tasks
- designing a dashboard with meaningful summary information
- creating pages that feel connected instead of isolated
- improving UX for real business workflows
- making the app look structured, trustworthy, and easy to scan

## What I Focused On

- building a more serious business-oriented SaaS interface
- improving dashboard and table/list design
- creating clearer relationships between app sections
- making the UI feel more complete and product-like
- improving visual hierarchy, spacing, and usability
- moving beyond a simple portfolio CRUD app

## Challenges

Some of the main challenges in this project were:

- balancing simplicity with product depth
- making the app feel realistic without overcomplicating the scope
- keeping the design clean while presenting more business data
- creating consistency across dashboard, tables, cards, forms, and detail pages

## Lessons Learned

This project helped me improve in several areas:

- structuring a more advanced multi-page React application

- thinking more like a product builder, not just a UI implementer
